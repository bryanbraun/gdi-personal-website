Github Pages

What is it?

It's free static site hosting provided by Github.

If anybody is unfamiliar with Github, then describe it in a sentence here.

Why Github Pages?

  • Uses developer workflows
  • Full control of your design & content
  • Ease of migration
  • Embraces openness
  • Large Plugin Ecosystem
  • Low Complexity
    • Secure
    • Scalable
    • Backups built in
    • No comment spam
  • Free
  • Uses Developer workflows: Git, Github, Ruby, Markdown, Frontend Code
  • Hosted services (squarespace, etc) often trade convenience for control.
  • Plugins: RSS, Sitemaps, Syntax Highlighting, etc.
  • Lots of options seem close to free but they tend to increase over time and add up.

The Case Against Github Pages

  • You can't use it to experiment with Backend / Databases / Caching / Servers / Ops
  • Favors Jekyll, which you may not want (like if you aren't interested in Ruby, for example)
  • Some features must use 3rd party JS plugins
  • Favors control over convenience.
    • Not a strong theme ecosystem.

Missing conveniences

  • No Drag & Drop image uploads.
  • No WYSIWYG Editor. Uses HTML instead (or Markdown, if you go with Jekyll)
  • No Large high-quality theme ecosystem
  • No 3rd Party Only: Commenting, Upvotes, Ratings, [your cool idea]
